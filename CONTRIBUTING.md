# Contributing to OpenGRC Toolkit

Thank you for your interest in contributing to OpenGRC Toolkit. This project thrives on community contributions.

## How to Contribute

### Submitting New Content

1. **Fork** the repository
2. **Create a branch** for your contribution (`git checkout -b add-backup-policy`)
3. **Add your content** following our templates and conventions
4. **Submit a Pull Request** with a clear description

### Content Guidelines

#### Writing Style
- Use clear, professional language
- Avoid jargon where possible; define terms when necessary
- Write in active voice
- Keep sentences concise

#### Document Structure
All documents should include:
- **Title** - Clear, descriptive heading
- **Metadata** - Version, date, owner (use placeholders)
- **Purpose** - Why this document exists
- **Scope** - What it covers and doesn't cover
- **Content** - The main body
- **References** - Links to related documents

#### Placeholder Conventions
Use these standard placeholders:
- `[ORGANIZATION NAME]` - Company/client name
- `[DATE]` - Effective or revision date
- `[VERSION]` - Document version number
- `[OWNER]` - Document owner role/title
- `[DEPARTMENT]` - Relevant department name
- `<!-- CUSTOMIZE: note -->` - HTML comments for customization guidance

### File Naming
- Use lowercase with hyphens: `access-control-policy.md`
- Be descriptive but concise
- Follow existing patterns in each directory

### Linking
- Use relative links: `[Policy](../policies/information-security-policy.md)`
- Ensure all links work from the file's location
- Update index files when adding new documents

## What We're Looking For

### High Priority
- ISO 27001:2022 control-specific procedures
- Risk assessment templates and methodologies
- Incident response playbooks
- NIST CSF 2.0 mappings
- SOC 2 trust service criteria documentation

### Always Welcome
- Improvements to existing content
- Typo fixes and clarifications
- Framework cross-references
- Real-world lessons learned (anonymized)
- Translations

## Quality Standards

Before submitting, ensure your contribution:
- [ ] Follows the document structure guidelines
- [ ] Uses consistent placeholder conventions
- [ ] Contains no proprietary or confidential information
- [ ] Has been spell-checked
- [ ] Links to related documents where appropriate
- [ ] Includes practical, actionable content

## Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Assume good intentions
- Help others learn

## Questions?

Open an issue for questions or suggestions about the contribution process.
