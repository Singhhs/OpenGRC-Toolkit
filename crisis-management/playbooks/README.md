# Incident Response Playbooks

Step-by-step guides for responding to specific incident types.

## Available Playbooks

| Playbook | Scenario |
|----------|----------|
| [Ransomware](./ransomware-playbook.md) | Ransomware infection and encryption |
| [Data Breach](./data-breach-playbook.md) | Unauthorized data access or exfiltration |
| [Phishing/BEC](./phishing-playbook.md) | Phishing attacks and business email compromise |
| [DDoS](./ddos-playbook.md) | Distributed denial of service attacks |
| [Insider Threat](./insider-threat-playbook.md) | Malicious or negligent insider activity |
| [Lost/Stolen Device](./lost-device-playbook.md) | Lost or stolen laptops, phones, media |
| [Third-Party Breach](./third-party-breach-playbook.md) | Vendor or partner security incident |
| [Malware](./malware-playbook.md) | General malware infection |
| [Account Compromise](./account-compromise-playbook.md) | Compromised user credentials |

## Playbook Structure

Each playbook follows a consistent structure:

1. **Overview** - What this playbook covers
2. **Detection Indicators** - How to identify this incident type
3. **Severity Assessment** - How to classify severity
4. **Response Steps** - Detailed actions by phase
5. **Communication** - Who to notify and when
6. **Recovery** - Restoration procedures
7. **Post-Incident** - Documentation and improvements

## How to Use

1. Identify the incident type
2. Open the corresponding playbook
3. Follow the steps in order
4. Document all actions taken
5. Adapt as needed for your specific situation
