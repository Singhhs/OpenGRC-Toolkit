## 1. Purpose

This procedure defines the process for delivering, tracking, and measuring security awareness training at [ORGANIZATION NAME] to ensure all personnel understand their security responsibilities.

## 2. Scope

This procedure applies to:
- All employees (permanent, temporary, part-time)
- Contractors and consultants
- Third-party personnel with system access
- New hires and role changes
- All training delivery methods

## 3. Training Program Overview

### 3.1 Training Categories

| Category | Audience | Frequency | Duration |
|----------|----------|-----------|----------|
| **General Awareness** | All personnel | On hire + Annual | 30-60 min |
| **Phishing Awareness** | All personnel | Quarterly simulations | 5-10 min |
| **Role-Specific** | Specific roles | On assignment + Annual | 30-60 min |
| **Privileged User** | IT administrators | On assignment + Annual | 60-90 min |
| **Developer Security** | Developers | On hire + Annual | 60-120 min |
| **Executive Briefing** | Senior management | Annual | 30 min |
| **Incident Response** | IR team members | Semi-annual | 60 min |

### 3.2 Core Topics

| Topic | Included In |
|-------|-------------|
| Security policies overview | General Awareness |
| Password security | General Awareness |
| Phishing and social engineering | General Awareness, Phishing |
| Data classification and handling | General Awareness |
| Clean desk and physical security | General Awareness |
| Mobile device security | General Awareness |
| Remote work security | General Awareness |
| Incident reporting | General Awareness |
| Secure coding practices | Developer Security |
| Privileged access management | Privileged User |
| Compliance requirements | Role-Specific |

## 4. Procedure

### Phase 1: Planning

**Step 1: Annual Training Plan**

At start of each year:
1. Review previous year's training effectiveness
2. Identify new threats and topics
3. Update training content as needed
4. Set training schedule
5. Define success metrics
6. Obtain budget approval

**Step 2: Content Development/Update**

For each training module:
- [ ] Review for current threats
- [ ] Update policy references
- [ ] Incorporate lessons from incidents
- [ ] Add relevant examples
- [ ] Update quizzes/assessments
- [ ] Obtain content approval

**Step 3: Training Calendar**

Create annual calendar:

| Month | Training Activity |
|-------|-------------------|
| January | Annual awareness training launch |
| February | Phishing simulation #1 |
| March | Role-specific training |
| April | Developer security training |
| May | Phishing simulation #2 |
| June | Mid-year metrics review |
| July | New threat briefings |
| August | Phishing simulation #3 |
| September | Privileged user training |
| October | Cybersecurity Awareness Month activities |
| November | Phishing simulation #4 |
| December | Year-end review and planning |

### Phase 2: New Hire Training

**Step 4: Onboarding Security Training**

Within first week of employment:

| Day | Activity |
|-----|----------|
| Day 1 | Security policy acknowledgment |
| Day 1-3 | General awareness training (online) |
| Week 1 | Complete training assessment |
| Week 1 | Account access provisioned after training complete |

**Step 5: Training Assignment**

1. HR notifies Security of new hire
2. Training account created
3. Training modules assigned
4. Deadline set (typically 5 business days)
5. Reminders sent if not complete

**Step 6: Completion Verification**

- [ ] Training completed
- [ ] Assessment passed (minimum score: [80%])
- [ ] Policy acknowledgment signed
- [ ] Record updated in training system

### Phase 3: Annual Awareness Training

**Step 7: Launch Annual Training**

1. Notify all personnel of training requirement
2. Provide clear deadline
3. Explain how to access training
4. Include consequence of non-completion

**Step 8: Training Delivery**

Delivery methods:
| Method | Use Case |
|--------|----------|
| Online self-paced | Primary method for all |
| Live webinar | New content, Q&A sessions |
| In-person | High-risk roles, sensitive topics |
| Video | Supplementary content |
| Newsletters | Ongoing awareness |

**Step 9: Track Completion**

Monitor weekly:
- Completion percentage
- Outstanding personnel
- Assessment scores

**Step 10: Reminders and Escalation**

| Timeline | Action |
|----------|--------|
| 2 weeks before deadline | First reminder to incomplete |
| 1 week before deadline | Second reminder |
| Deadline | Escalate to manager |
| 1 week after deadline | Escalate to HR/executive |
| Continued non-compliance | Disciplinary process |

### Phase 4: Phishing Simulations

**Step 11: Plan Simulation**

For each simulation:
- Select phishing template
- Define target population
- Schedule send date/time
- Configure reporting page
- Set up metrics collection

**Step 12: Execute Simulation**

1. Send simulated phishing emails
2. Monitor click rates and reports
3. Provide immediate feedback to those who click
4. Track reporters (positive behavior)

**Step 13: Report Results**

After each simulation:
- Click rate (target: <5%)
- Report rate (target: >20%)
- Comparison to previous simulations
- Department breakdown
- Repeat clickers identified

**Step 14: Follow-Up Training**

For employees who clicked:
1. Immediate redirect to training
2. Assignment of additional phishing module
3. Track completion
4. Flag repeat offenders for manager discussion

### Phase 5: Role-Specific Training

**Step 15: Identify Training Needs**

Based on job function:

| Role | Additional Training Required |
|------|------------------------------|
| Finance/Accounting | BEC/wire fraud, financial data handling |
| HR | PII protection, employee data handling |
| IT Administrators | Privileged access, secure configuration |
| Developers | Secure coding, OWASP Top 10 |
| Executives | Targeted attacks, travel security |
| Customer Service | Social engineering, data verification |
| Legal/Compliance | Regulatory requirements, eDiscovery |

**Step 16: Assign and Track**

1. Identify personnel in each role category
2. Assign appropriate training modules
3. Set completion deadlines
4. Monitor and follow up

### Phase 6: Measurement and Reporting

**Step 17: Collect Metrics**

Track the following:

| Metric | Target | Frequency |
|--------|--------|-----------|
| Training completion rate | >95% | Monthly |
| Average assessment score | >85% | Quarterly |
| Phishing click rate | <5% | Per simulation |
| Phishing report rate | >20% | Per simulation |
| Repeat clickers | <2% | Quarterly |
| Security incidents (human error) | Decreasing | Quarterly |
| Help desk security questions | Tracked | Monthly |

**Step 18: Report to Management**

| Report | Audience | Frequency |
|--------|----------|-----------|
| Training dashboard | IT Security | Weekly |
| Completion summary | Department managers | Monthly |
| Awareness metrics | CISO/IT Director | Quarterly |
| Annual assessment | Executive/Board | Annually |

**Step 19: Program Evaluation**

Annually assess:
- Training effectiveness (behavior change)
- Content relevance
- Delivery methods
- Employee feedback
- Incident correlation
- Industry benchmarks

### Phase 7: Continuous Improvement

**Step 20: Gather Feedback**

After training:
- Post-training surveys
- Help desk feedback
- Manager input
- Incident analysis

**Step 21: Update Program**

Based on:
- New threats and attack techniques
- Policy changes
- Regulatory updates
- Audit findings
- Feedback received
- Industry best practices

### 5.2 Assessment Requirements

## 6. Non-Compliance Management

| Offense | Action |
|---------|--------|
| First non-completion | Manager notification, deadline extension |
| Second non-completion | HR notification, written warning |
| Third non-completion | Access restriction, disciplinary action |
| Repeat phishing failures | Additional training, manager meeting |
| Policy violation | Per disciplinary policy |

- Assessment scores
- Policy acknowledgments
- Phishing simulation results
- Feedback and surveys

**Retention:** Duration of employment + [3] years
