## 1. Purpose

This procedure defines the process for planning, conducting, and reporting internal audits of [ORGANIZATION NAME]'s Information Security Management System (ISMS) in accordance with ISO 27001.

## 2. Scope

Applies to:
- All ISMS processes and controls
- All departments within ISMS scope
- All locations covered by the ISMS
- ISO 27001 clauses and Annex A controls

| Q3 | [Areas] | [Departments] |

### 3.2 Audit Frequency

| Audit Type | Frequency |
|------------|-----------|
| Full ISMS audit | Annual |
| Process-specific audits | Per schedule |
| Follow-up audits | As needed |
| Special audits | Triggered by events |

### 3.3 Audit Triggers

Additional audits may be triggered by:
- Security incidents
- Significant changes to ISMS
- Management request
- Regulatory requirements
- Previous audit findings

## 4. Audit Procedure

### Phase 1: Audit Planning

#### Step 1.1: Develop Audit Plan

**Responsibility:** Lead Auditor / Audit Manager

**Timeline:** 4 weeks before audit

1. [ ] Define audit objectives
2. [ ] Determine audit scope:
   - ISO 27001 clauses to audit
   - Annex A controls to audit
   - Processes to examine
   - Departments/locations
3. [ ] Select audit team
4. [ ] Verify auditor independence
5. [ ] Schedule audit dates
6. [ ] Allocate resources

**Audit Plan Contents:**
| Element | Details |
|---------|---------|
| Audit ID | AUD-[YEAR]-[NUMBER] |
| Audit Objectives | |
| Scope | |
| Criteria | ISO 27001:2022, ISMS policies |
| Audit Team | |
| Audit Dates | |
| Auditee Contacts | |

---

#### Step 1.2: Notify Auditees

**Responsibility:** Lead Auditor

**Timeline:** 3 weeks before audit

1. [ ] Send audit notification to:
   - Department managers
   - Process owners
   - Key personnel
2. [ ] Include:
   - Audit dates and times
   - Scope and objectives
   - Documents to prepare
   - Interview schedule
   - Audit team members
3. [ ] Confirm availability
4. [ ] Address any concerns

**Notification Template:**
```
Subject: Internal ISMS Audit Notification - [DATE]

Dear [NAME],

This is to notify you of an upcoming internal audit of the
Information Security Management System.

Audit ID: [ID]
Dates: [DATES]
Scope: [SCOPE]
Lead Auditor: [NAME]

Please have the following available:
- [Documents/records]
- [Personnel for interviews]

Contact [NAME] with any questions.
```

---

#### Step 1.3: Prepare Audit

**Responsibility:** Audit Team

**Timeline:** 2 weeks before audit

1. [ ] Review relevant documentation:
   - ISMS policies and procedures
   - Previous audit reports
   - Risk assessment and treatment
   - Statement of Applicability
   - Incident reports
   - Management review minutes
2. [ ] Prepare audit checklist
3. [ ] Develop interview questions
4. [ ] Prepare audit working papers
5. [ ] Confirm logistics

---

### Phase 2: Conducting the Audit

#### Step 2.1: Opening Meeting

**Responsibility:** Lead Auditor

**Timeline:** Start of audit

**Agenda:**
1. [ ] Introductions
2. [ ] Confirm audit scope and objectives
3. [ ] Review audit schedule
4. [ ] Explain audit methodology
5. [ ] Confirm confidentiality
6. [ ] Explain finding categories
7. [ ] Questions and answers

**Attendees:**
- Audit team
- Department managers
- Process owners
- ISMS representative

---

#### Step 2.2: Evidence Collection

**Responsibility:** Audit Team

**Timeline:** During audit period

**Methods:**

| Method | Use For |
|--------|---------|
| Document review | Policies, procedures, records |
| Interviews | Understanding, verification |
| Observation | Process execution, physical controls |
| Testing | Control effectiveness |
| Sampling | Large populations |

**Evidence Types:**
- Documented evidence (records, logs)
- Statements from personnel
- Observations of activities
- Test results

**Documentation:**
1. [ ] Record evidence on working papers
2. [ ] Note source and date
3. [ ] Cross-reference to criteria
4. [ ] Obtain copies of key evidence
5. [ ] Note any limitations

---

#### Step 2.3: Evaluate Findings

**Responsibility:** Audit Team

**Timeline:** Throughout audit

For each audit area:
1. [ ] Compare evidence to criteria
2. [ ] Identify conformities
3. [ ] Identify nonconformities
4. [ ] Classify findings
5. [ ] Discuss with auditee
6. [ ] Verify understanding

**Finding Categories:**

| Category | Definition |
|----------|------------|
| **Major Nonconformity** | Absence or total breakdown of required process/control; significant risk |
| **Minor Nonconformity** | Isolated lapse; partial implementation; low risk |
| **Observation** | Not a nonconformity but improvement opportunity |
| **Positive Finding** | Noteworthy good practice |

---

#### Step 2.4: Daily Debrief (for multi-day audits)

**Responsibility:** Lead Auditor

**Timeline:** End of each audit day

1. [ ] Review day's findings with team
2. [ ] Brief auditee on preliminary findings
3. [ ] Clarify any misunderstandings
4. [ ] Confirm next day's schedule
5. [ ] Address any concerns

---

#### Step 2.5: Closing Meeting

**Responsibility:** Lead Auditor

**Timeline:** End of audit

**Agenda:**
1. [ ] Thank participants
2. [ ] Restate audit scope
3. [ ] Present audit findings:
   - Major nonconformities
   - Minor nonconformities
   - Observations
   - Positive findings
4. [ ] Explain next steps
5. [ ] Discuss timelines
6. [ ] Questions and clarifications

**Attendees:**
- Same as opening meeting
- Senior management (recommended)

---

### Phase 3: Reporting

#### Step 3.1: Draft Audit Report

**Responsibility:** Lead Auditor

**Timeline:** Within 1 week of audit completion

**Report Contents:**

1. **Executive Summary**
   - Audit conclusion
   - Summary of findings
   - Recommendations

2. **Audit Details**
   - Audit ID and dates
   - Scope and objectives
   - Criteria used
   - Audit team
   - Auditees

3. **Findings**
   - Each finding with:
     - Finding ID
     - Category
     - Description
     - Evidence
     - Criteria reference
     - Risk/impact

4. **Observations and Opportunities**

5. **Positive Findings**

6. **Appendices**
   - Audit checklist
   - Attendance records
   - Evidence list

---

#### Step 3.2: Review and Finalize Report

**Responsibility:** Lead Auditor / Audit Manager

**Timeline:** Within 2 weeks of audit

1. [ ] Review draft for accuracy
2. [ ] Verify findings supported by evidence
3. [ ] Share draft with auditees for factual review
4. [ ] Address any factual corrections
5. [ ] Finalize report
6. [ ] Obtain audit manager approval

---

#### Step 3.3: Distribute Report

**Responsibility:** Audit Manager

**Timeline:** Within 2 weeks of audit

Distribute to:
- [ ] Auditee management
- [ ] ISMS owner/manager
- [ ] CISO
- [ ] Senior management
- [ ] Audit file

---

### Phase 4: Follow-Up

#### Step 4.1: Corrective Action

**Responsibility:** Auditee / Process Owner

**Timeline:** As defined

For each nonconformity:
1. [ ] Acknowledge finding
2. [ ] Determine root cause
3. [ ] Define corrective action
4. [ ] Assign responsibility
5. [ ] Set target date
6. [ ] Submit corrective action plan

**Corrective Action Timeframes:**

| Finding Type | Response | Completion |
|--------------|----------|------------|
| Major NC | 5 business days | 30 days |
| Minor NC | 10 business days | 60 days |
| Observation | 30 days | 90 days |

---

#### Step 4.2: Verify Corrective Actions

**Responsibility:** Lead Auditor

**Timeline:** After target date

1. [ ] Review corrective action evidence
2. [ ] Verify effectiveness
3. [ ] Close finding if effective
4. [ ] Escalate if not effective
5. [ ] Update audit tracking

---

#### Step 4.3: Follow-Up Audit (if needed)

**Responsibility:** Audit Team

**Timeline:** For major nonconformities

1. [ ] Schedule follow-up audit
2. [ ] Focus on nonconformity areas
3. [ ] Verify corrective actions implemented
4. [ ] Verify effectiveness
5. [ ] Report results

## 5. Auditor Requirements

### Auditor Qualifications

| Requirement | Lead Auditor | Auditor |
|-------------|--------------|---------|
| ISO 27001 knowledge | Required | Required |
| Audit training | ISO 27001 Lead Auditor | Internal audit training |
| Audit experience | 3+ audits | 1+ audit |
| Independence | Cannot audit own work | Cannot audit own work |

### Auditor Independence

Auditors must not audit:
- Their own work
- Their own department
- Areas they have direct responsibility for
- Areas where they have conflicts of interest
