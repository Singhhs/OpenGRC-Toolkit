## 1. Purpose

This procedure defines the process for receiving, validating, and responding to data subject requests (DSRs) in compliance with data protection regulations including GDPR at [ORGANIZATION NAME].

## 2. Scope

This procedure applies to:
- All requests from data subjects exercising their rights
- All personal data held by [ORGANIZATION NAME]
- All employees who may receive or process DSRs
- All systems containing personal data

## 3. Data Subject Rights

| Right | Description | Applies When |
|-------|-------------|--------------|
| **Right to be Informed** | Know how data is used | Always (via privacy notice) |
| **Right of Access** | Obtain copy of personal data | Subject to exemptions |
| **Right to Rectification** | Correct inaccurate data | Data is incorrect |
| **Right to Erasure** | Delete personal data | Various conditions apply |
| **Right to Restrict Processing** | Limit how data is used | Various conditions apply |
| **Right to Data Portability** | Receive data in portable format | Consent/contract basis, automated |
| **Right to Object** | Object to certain processing | Direct marketing, legitimate interests |
| **Rights re: Automated Decisions** | Human review of automated decisions | Automated decision-making |

## 4. Response Timeframes

| Regulation | Standard Timeframe | Extension |
|------------|-------------------|-----------|
| GDPR | 1 month (30 days) | +2 months for complex requests |
| CCPA | 45 days | +45 days with notice |
| Other | Per applicable law | As permitted |

**Note:** Time runs from date of receipt (or identity verification, if required).

## 5. Procedure

### Phase 1: Receipt and Logging

**Step 1: Receive Request**

Requests may be received via:
- Email: [privacy@organization.com]
- Web form: [privacy portal URL]
- Post: [mailing address]
- In person: [reception/HR]
- Phone: [with written follow-up]

**Step 2: Log the Request**

Record in DSR tracking system:

| Field | Information |
|-------|-------------|
| DSR ID | Unique identifier |
| Date Received | Date request received |
| Received Via | Channel of receipt |
| Requestor Name | Name of data subject |
| Contact Details | Email, phone, address |
| Request Type | Access, erasure, etc. |
| Description | Details of request |
| Deadline | Response due date |
| Status | Open |
| Handler | Assigned processor |

**Step 3: Acknowledge Receipt**

Send acknowledgment within 2 business days:
- Confirm request received
- Provide DSR reference number
- Explain next steps
- Set expectations for timeline
- Request additional info if needed

### Phase 2: Validation

**Step 4: Verify Identity**

Before processing, verify requestor identity:

| Risk Level | Verification Method |
|------------|---------------------|
| Low (known customer) | Email from registered account |
| Medium | Security questions + email confirmation |
| High | Photo ID + proof of address |
| Agent/Representative | Written authorization + ID |

**Verification checklist:**
- [ ] Identity confirmed
- [ ] Requestor is the data subject (or authorized agent)
- [ ] Sufficient information to locate records
- [ ] Verification documented

**Step 5: Clarify Request (if needed)**

If request is unclear:
- Contact requestor for clarification
- Document communication
- Pause deadline clock until clarified

**Step 6: Assess Request Validity**

Determine:
- Which right(s) being exercised
- Whether right applies in this case
- Whether exemptions apply
- Whether request is manifestly unfounded or excessive

**Exemption considerations:**
| Exemption | Examples |
|-----------|----------|
| Legal proceedings | Data needed for legal claims |
| Legal obligations | Required to retain by law |
| Public interest | Journalism, research |
| Third-party rights | Would disclose others' data |
| Trade secrets | Commercial confidentiality |
| Manifestly unfounded | Harassment, repeated requests |

### Phase 3: Processing

**Step 7: Locate Personal Data**

Search all systems for requestor's data:

| System Category | Systems to Search |
|-----------------|-------------------|
| Core business | CRM, ERP, HR systems |
| Communications | Email, chat logs |
| Marketing | Marketing automation, analytics |
| Support | Help desk, call recordings |
| Finance | Billing, payment systems |
| Physical | Paper files, archives |
| Backups | Consider if accessible |

**Step 8: Process by Request Type**

#### Access Request (SAR)

1. Compile all personal data held
2. Include supplementary information:
   - Processing purposes
   - Categories of data
   - Recipients
   - Retention periods
   - Rights information
   - Source of data
3. Redact third-party personal data
4. Format for readability

#### Rectification Request

1. Verify correct information provided
2. Update records in all systems
3. Notify third parties if data was shared
4. Document changes made

#### Erasure Request

1. Assess if erasure grounds apply:
   - Consent withdrawn
   - No longer necessary
   - Objects to processing
   - Unlawfully processed
   - Legal obligation to erase
   - Child's data
2. Check for exemptions (legal hold, etc.)
3. Delete from all systems
4. Notify third parties if shared
5. Document what was deleted

#### Restriction Request

1. Assess if restriction grounds apply:
   - Accuracy contested
   - Processing unlawful, subject prefers restriction
   - No longer needed but subject needs for legal claims
   - Pending objection verification
2. Mark data as restricted
3. Stop processing (except storage)
4. Document restriction

#### Portability Request

1. Verify applies (consent/contract + automated)
2. Extract data in structured format (CSV, JSON)
3. Include only data provided by subject
4. Provide securely

#### Objection

1. Assess grounds for objection
2. For direct marketing: stop immediately
3. For legitimate interests: assess if compelling grounds override
4. Document decision

**Step 9: Quality Review**

Before sending response:
- [ ] All systems searched
- [ ] Data complete and accurate
- [ ] Third-party data redacted
- [ ] Response clearly explained
- [ ] Reviewed by second person (for complex requests)

### Phase 4: Response

**Step 10: Prepare Response**

Response should include:
- Reference to original request
- Summary of action taken
- Data provided (for access requests)
- Explanation if request refused/limited
- Information about complaints to supervisory authority

**Step 11: Deliver Response**

| Delivery Method | Use When |
|-----------------|----------|
| Secure email/portal | Standard method |
| Encrypted email | Sensitive data |
| Recorded post | Physical documents, ID verification |
| Hand delivery | High sensitivity |

**Step 12: Document Completion**

Update DSR log:
- Response date
- Response method
- Summary of response
- Any data provided
- Refusals and reasons
- Close request

### Phase 5: Special Situations

**Refusing a Request**

If refusing (wholly or partly):
1. Document valid reason for refusal
2. Inform requestor of:
   - Reason for refusal
   - Right to complain to supervisory authority
   - Right to judicial remedy
3. Retain documentation

**Complex Requests**

If extension needed:
1. Must notify within original deadline
2. Explain reasons for delay
3. Provide new deadline (max +2 months)
4. Document justification

**Third-Party Requests**

For requests by agents/representatives:
1. Verify agent's identity
2. Confirm authorization from data subject
3. Retain authorization documentation
4. Respond to authorized agent

**Requests Involving Others' Data**

1. Identify third-party personal data
2. Assess if disclosure would adversely affect
3. Redact or seek third-party consent
4. Document decision

## 6. Response Templates

### Acknowledgment Template

```
Subject: Data Subject Request Received - [DSR-XXXX]

Dear [Name],

Thank you for your request received on [date].

Your request has been logged with reference number DSR-XXXX.

We will respond within [30 days / timeframe]. If we need
additional time, we will let you know.

[If verification needed:]
To protect your privacy, please provide [verification details].

If you have questions, contact [privacy@organization.com].

Regards,
[Privacy Team]
```

### Response Template (Access)

```
Subject: Response to Your Data Subject Request - [DSR-XXXX]

Dear [Name],

Further to your request dated [date], please find enclosed
the personal data we hold about you.

[Summary of data categories]

The attached document contains your personal data in [format].
Please find below additional information about our processing:

Purpose of processing: [purposes]
Categories of recipients: [recipients]
Retention period: [periods]

If you believe any information is inaccurate, you may request
rectification. You also have the right to lodge a complaint
with [supervisory authority].

Regards,
[Privacy Team]
```

## 7. Escalation

| Situation | Escalate To |
|-----------|-------------|
| Complex legal questions | Legal / DPO |
| Refusal decisions | DPO approval |
| Requests from regulators | Legal + Executive |
| Potential data breach | DPO + Security |
| Complaints | DPO |

## 8. Metrics

Track and report:
- DSRs received (by type)
- Response time (average, compliance %)
- Requests refused (with reasons)
- Extensions used
- Complaints received

| **IT** | Search systems, extract data |
| **Legal** | Advise on exemptions, refusals |
| **Business Units** | Provide data, verify accuracy |
