# ISO 27001:2022 Resources

ISO/IEC 27001:2022 is the international standard for establishing, implementing, maintaining, and continually improving an Information Security Management System (ISMS).

## ISO 27001:2022 Structure

### Clauses (Management System Requirements)

| Clause | Title | Key Documents |
|--------|-------|---------------|
| 4 | Context of the Organization | [ISMS Scope](./policies/isms-scope.md), Stakeholder Analysis |
| 5 | Leadership | [Information Security Policy](../../policies/information-security-policy.md) |
| 6 | Planning | [Risk Assessment](../../risk-assessment/README.md), Statement of Applicability |
| 7 | Support | Training Records, Communication Plan |
| 8 | Operation | Operational Procedures, Risk Treatment |
| 9 | Performance Evaluation | Internal Audit, Management Review |
| 10 | Improvement | Corrective Actions, Continual Improvement |

### Annex A Controls (2022 Version)

The 2022 version reorganized controls into 4 themes with 93 controls:

| Theme | Controls | Description |
|-------|----------|-------------|
| Organizational (5) | 37 controls | Policies, roles, responsibilities |
| People (6) | 8 controls | HR security, awareness |
| Physical (7) | 14 controls | Physical and environmental security |
| Technological (8) | 34 controls | Technical security controls |

## Implementation Roadmap

### Phase 1: Foundation (Weeks 1-4)
- [ ] Define ISMS scope
- [ ] Establish security policy
- [ ] Assign roles and responsibilities
- [ ] Conduct initial risk assessment

### Phase 2: Risk Treatment (Weeks 5-8)
- [ ] Complete risk treatment plan
- [ ] Develop Statement of Applicability
- [ ] Implement priority controls
- [ ] Create core procedures

### Phase 3: Implementation (Weeks 9-16)
- [ ] Implement remaining controls
- [ ] Develop all required documentation
- [ ] Conduct security awareness training
- [ ] Establish monitoring processes

### Phase 4: Verification (Weeks 17-20)
- [ ] Conduct internal audit
- [ ] Perform management review
- [ ] Address nonconformities
- [ ] Prepare for certification audit

## Required Documents

ISO 27001 requires the following documented information:

### Mandatory Documents
1. Scope of the ISMS (4.3)
2. Information security policy (5.2)
3. Risk assessment process (6.1.2)
4. Risk treatment process (6.1.3)
5. Statement of Applicability (6.1.3 d)
6. Risk treatment plan (6.2)
7. Risk assessment results (8.2)
8. Risk treatment results (8.3)
9. Monitoring and measurement results (9.1)
10. Internal audit program and results (9.2)
11. Management review results (9.3)
12. Nonconformities and corrective actions (10.1)

### Mandatory Records
- Training records
- Monitoring results
- Audit results
- Management review minutes
- Corrective action records
