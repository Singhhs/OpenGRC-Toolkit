## GDPR Overview

### Key Principles (Article 5)

| Principle | Description |
|-----------|-------------|
| **Lawfulness, fairness, transparency** | Process data lawfully and transparently |
| **Purpose limitation** | Collect for specified, explicit purposes |
| **Data minimization** | Adequate, relevant, limited to necessary |
| **Accuracy** | Keep personal data accurate and up to date |
| **Storage limitation** | Keep only as long as necessary |
| **Integrity and confidentiality** | Appropriate security measures |
| **Accountability** | Demonstrate compliance |

### Data Subject Rights

| Right | Description |
|-------|-------------|
| Right to be informed | Know how data is used |
| Right of access | Obtain copies of data |
| Right to rectification | Correct inaccurate data |
| Right to erasure | Request deletion |
| Right to restrict processing | Limit processing |
| Right to data portability | Receive data in portable format |
| Right to object | Object to certain processing |
| Rights related to automated decision making | Human review of automated decisions |

### Key Requirements

| Article | Requirement |
|---------|-------------|
| 30 | Records of processing activities |
| 32 | Security of processing |
| 33-34 | Breach notification |
| 35 | Data protection impact assessment |
| 37-39 | Data Protection Officer |
| 44-49 | International transfers |

## GDPR vs. ISO 27001

| Aspect | GDPR | ISO 27001 |
|--------|------|-----------|
| Scope | Personal data | All information |
| Type | Regulation (mandatory in EU) | Voluntary standard |
| Focus | Privacy | Information security |
| Certification | No official certification | Formal certification |

ISO 27001 certification supports but does not guarantee GDPR compliance.
